一个问号引发的血案——小程序开发从学习到上线遇到那些坑
2019-03-28

速成小程序

把时间回拨到上上周，实验室接到一个小程序开发的项目，因为之前在简历上吹大创写过小程序的牛，天道好轮回，这个项目就交给我做了。

之前大创做的小程序基本是靠 ctrl c ctrl v 开发了一个录音程序半成品，而且还是两年前，基本啥都忘光了。

机会是留给有准备的人的，没有准备的人只能硬着头皮上了。

项目时间比较紧迫，大约一周的时间，刚装上ide，我就直接莽了起来，结果好端端的一个背景全屏适配，弄了差不多一个上午。总结教训：先动脑子，再写程序。

中午和潘琳吃饭倒苦水，正好她创新课也要写小程序，给我看了一个她找的网易云课堂的网课，感觉非常不错，怒斥一大洋买下。下午停止无脑开发，先补课，再写代码。

wxss里布局只看flex布局就行了，小程序对flex支持的挺不错，界面设计的时候先画格子，再布局。flex学习可以看网课，具体写的时候可以查阮一峰的博客。

页面和程序生命周期的概念官方文档基本讲的很清晰，这里记录一个文档里没提及的坑，小程序页面栈内最多存放 10 个页面，超过 10 个会无法跳转，页面跳转时用到的 wx.navigateTo() 和 wx.redirectTo() 的区别在于前者保留当前页面，跳转到应用内的某个页面,而后者则关闭当前页面再跳转。所以页面过多时建议使用 wx.redirectTo() 。

这次的项目是做一个会展社交类的app，潘琳给我看的教程其实特别贴合我的需求（手动比心❤），比如列表页中要把每一栏的用户封装成一个 navigator 组件再动态渲染，又比如 user 页要取得 recommendation 中的数据来显示，页面之间要用对象查询的方式传参。这些做到后面其实也都是一些机械性的布局和编写了，体力劳动胜过脑力劳动。

含混不清的页面逻辑

项目方一开始给了设计稿和api，大手一挥说：你去做吧。我看了半天也没看到页面的跳转逻辑，之后才草草写了一个word发给我。
他们的程序是：许多参加会展的人缺乏相互勾兑的平台，于是他们要做一个小程序，展示用户的信息，互相看对眼的用户可以发送 invitation，邀请得到通过后，系统会自动为用户安排私人会面的时间和地点，虽然是社交软件，但是丝毫不涉及到什么别的 im通讯。

设计图上在recommendation页面是有logout功能的，因为小程序右上角有固定的图标，和这个logout本来的布局重叠了，我问项目方该咋放，他们才告诉我后端没做这个功能，用户登陆进来了就别想出去了……
其实前端也能实现，当前的登陆只是从服务器返回一个token，不涉及微信账号的关联，所以我直接在前端登陆后本地缓存token，登出时清除缓存，然后在index页面的onShow函数中检查token是否为null实现了登出功能。

加量不加价，只收一样钱。

似是而非的备案域名

这个项目之所以催的紧，是因为当周的周末就有展会要使用。所以不仅要开发，也要提交审核，上线。

微信在小程序内部发起网络请求这部分限制的很严格，用到的域名必须经过icp备案，而且限制多多，不说了，上图。

意识到这个问题的时候已经周四了，项目方团队是在加拿带，我寻思十有八九没有备案，问了果真如此。查了一下，icp备案至少要两周，麻烦的很，于是求助万能的淘宝，无果。项目方最后花钱找了一个备案过的域名，把服务器绑定到了人家的二级域名上，这才完事儿…… 同时吸取到一个教训，DNS最长生效时间是48个小时，即使这边配置好了，全网同步的时间可能要两天，这个时间不受自己控制，也没有办法催，我晚上的时候一直测试不成功，半夜爬起来才能访问。并且似乎小程序里面的dns用的是腾讯自己的dns服务器（119.29.29.29），即使他们在加拿大能访问到那个二级域名，也没办法在小程序中使用。

一个问号引起的神坑

写到后面基本没遇上什么特别难调整的bug，因为本来就是做前端，后端api和数据库都设计好了。但是最后两天遇到的一个bug必须记录一下，页面之前传参的时候千万小心问号…..

前面提到从 recommendation 跳转到 user 页面时要穿参，user页才能渲染。这里我本来用的是将需要传递的参数拼接到 itemurl + ? 之后做的：

虽然丑，但是应该是能用的。

可是我调试来调试去，总是传了几个参数，后面就截断……百思不得其解,console.log打了无数个戳子都没有用。

我一开始还以为是传递参数长度有限制，测试了一下不应该，结果把几个参数的顺序调换了一下，出现截断的位置改变了。这才意识到，应该是传递参数里面的内容引起了阶段。

再一看果然是，参数里面 q2 的内容有一个 “？” ，这个url直接被问号截断了，后面的参数都没传过去。

这个问题的解决方案是传参的时候别直接传内容，而是先把对象中的 url 单独encode，再转为字符串

传递到目的页后，再使用

还原为该对象，并解码。

这样就不会被截断了，除了”?”,”、” “&” 也都不行。

这个bug调了很久，有一点难受……
